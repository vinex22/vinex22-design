# SFX Library · vinex22-design

> All SFX are generated by the ElevenLabs Sound Generation API — Apple-keynote audio quality.
> Production-grade SFX library covering animation / demo / product-demo scenes.

**Asset location**: `assets/sfx/<category>/<name>.mp3` (not bundled — see "Note on binary audio assets" at the end)
**Total**: 37 SFX (30 batch-generated + 7 v7b kept)
**Generation model**: ElevenLabs Sound Generation API (`prompt_influence: 0.4`)
**Audio quality**: 44.1 kHz MP3, Apple-keynote clarity, no extra reverb

## Usage standards

### Volume guidance (from `audio-design-rules.md` dual-track audio)
- **SFX main track**: `1.0` (no attenuation)
- **BGM background**: `0.4 ~ 0.5` (SFX cuts through clearly)
- **Multi-SFX overlay**: use `amix=inputs=N:duration=longest:normalize=0` to keep dynamic range

### ffmpeg overlay templates
```bash
# Single SFX aligned to a time point:
ffmpeg -i video.mp4 -itsoffset 2.5 -i sfx/ui/click.mp3 \
  -filter_complex "[0:a][1:a]amix=inputs=2:duration=longest:normalize=0[a]" \
  -map 0:v -map "[a]" output.mp4

# Multiple SFX + BGM:
ffmpeg -i video.mp4 \
  -itsoffset 1.0 -i sfx/transition/whoosh-fast.mp3 \
  -itsoffset 1.6 -i sfx/impact/logo-reveal-v2.mp3 \
  -i bgm.mp3 \
  -filter_complex "[3:a]volume=0.4[bgm];[0:a][1:a][2:a][bgm]amix=inputs=4:normalize=0[a]" \
  -map 0:v -map "[a]" output.mp4
```

### Selection decision tree
1. **Tactile action** (typing / click / swipe) → `keyboard/` or `ui/`
2. **Element entry / exit** → `transition/`
3. **Container-layer ops** (cards / modals) → `container/`
4. **State feedback** (success / failure / notify) → `feedback/`
5. **Progress / time passage** → `progress/`
6. **Brand landing / important moment** → `impact/`
7. **AI magic / transform** → `magic/`
8. **Command line / code demo** → `terminal/`

### Avoid stacked sounds
- At a single time point, max 2 concurrent SFX
- BGM under 0.3 → can do 3
- During brand impact, mute other SFX (give 0.2s of silence before the hit)

## Prompt-writing principles (reusable)

Reference style: `apple keynote, tight, minimal, no reverb unless ambient, crisp, elegant`

**Three ingredients of a good prompt**:
1. **Sound physics**: what object, what action ("mechanical keyboard single key press")
2. **Texture / style constraint**: apple-style / ios-style / cinematic / retro
3. **Negative exclusions**: no reverb / clean studio / minimal

❌ "click sound"
✅ "crisp ui button click, clean modern interface sound, apple-style, high pitched"

❌ "magic"
✅ "bright twinkling stars sound, high pitched glittery chime, fairy dust"

## Note on binary audio assets

Per the repository's lightweight policy, the actual `.mp3` files are **not bundled**. To populate `assets/sfx/`:

1. Sign up for [ElevenLabs](https://elevenlabs.io) and get a Sound Generation API key.
2. For each SFX listed above, call the Sound Generation endpoint with the prompt in the "Prompt key points" column and `prompt_influence: 0.4`.
3. Save the result as `assets/sfx/<category>/<name>.mp3`.

You can also substitute any equivalent royalty-free SFX library (e.g. Freesound, Zapsplat). The recipes in `audio-design-rules.md` will work as long as the file paths match.
